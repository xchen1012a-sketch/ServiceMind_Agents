# 输出协议

默认使用简体中文输出计划、报告、解释、Handoff 和 AI 维护文档。代码标识符、命令、日志、协议字段与引用原文保持原样；不得因项目内容为英文而自行切换语言，只有用户明确要求时才允许当前任务例外。

默认短版。只有高风险、接入、审计、失败、用户要求详细说明时，才输出长版。

输出给用户必须抓重点：先结论，再验证，再风险/未证实项；不要用过程流水账占篇幅。

## 分级输出

| 等级 | 输出要求 |
| --- | --- |
| L0 | 1-3 行，只说结论 |
| L1 | 最多 5 行：完成项 + 验证 |
| L2 | 三段：完成情况、验证、注意 |
| L3 | 完整交付报告：改动、验证、风险、回滚 |
| L4 | 完整报告，可用窄表格；仍避免流水账 |

## 终端友好表格

表格可以用，但必须可读：

- 表格最多 3 列。
- 每个单元格只放短信息；超过 30 字改用编号列表。
- 风险清单优先用编号列表，表格只用于短字段对比。
- 证据、影响、建议不要挤在同一格里。

推荐风险格式：

```text
主要风险：

1. 风险标题
   - 影响：...
   - 依据：...
   - 建议：...
```

## 默认短版模板

```text
完成情况：
- ...

验证：
- ...

注意：
- ...

上下文：Lx；已读取：...；未读取：...；升级原因：...；未证实：无/...
```

## 禁止项

- 禁止流水账式复述每一步工具调用。
- 禁止罗列所有读过的规范文件。
- 禁止用大段解释替代结论。
- 禁止在无风险小改动里输出长篇背景。
- 禁止宽表格：超过 3 列或单元格长句必须改写。
- 禁止把未验证、未读取或无法证明的判断写成确定结论；必须压缩成一行 `未证实：...`。

## 升级为长版的条件

- 本次实际修改公开契约、鉴权/权限边界、数据库 schema/迁移、敏感数据或安全边界；项目仅存在 API/DB 不升级。
- 验证失败或有未验证项。
- 出现 `[⚠️ 冲突]`。
- 用户要求详细说明、审计或交接。
