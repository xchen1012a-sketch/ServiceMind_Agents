# 通用问题诊断

先诊断，再修复。避免一上来猜代码、改代码。

## 启动

> 假定 AI-START.md 启动协议已完成。上下文路由按 AI-START.md 执行，本 Skill 不建立第二套路由。

1. 收集现象、报错、日志、输入、环境、最近改动和复现步骤。
2. 区分事实、假设、未知项。
3. 建立最小复现或最小失败命令。
4. 缩小可疑范围，避免同时改多个方向。
5. 给出根因判断、证据和最小修复方案。

## 诊断顺序

- 是否能稳定复现。
- 失败发生在哪一层：输入、配置、依赖、环境、业务逻辑、数据、权限、外部系统。
- 最近变更是否相关。
- 是否有同类模块或历史实现可对照。
- 是否能用一条最小测试或命令证明根因。

## 输出

```text
结论：...

证据：
- ...

根因：
- confirmed / likely / unknown

最小修复建议：
- ...

是否需要交给 dev-implementation：
- 是/否，原因...
```

未复现的问题必须标注为“未复现”，不得写成已确认根因。
