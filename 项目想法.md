 项目名：企业工单与知识库多 Agent 处理平台（ServiceMind Agents）

  一句话定位：

> 一个面向企业客服/运营/IT 支持场景的多 Agent 系统，可以自动理解工单、检索知识库、调用业务工具、生成处理方案、升级人工
> 审批，并沉淀知识与评估效果。

  这个项目很适合写简历，因为它不是 Demo，而是能体现完整 AI 工程能力。

  核心功能

1. 工单接入
   支持用户提交问题，例如退款、账号异常、系统故障、合同问题、数据查询等。

2. 多 Agent 编排
   用 LangGraph 做状态流转：
   
   - 分类 Agent：判断工单类型、优先级、风险等级
   - RAG Agent：从企业知识库检索资料
   - Tool Agent：调用数据库、CRM、订单系统、日志系统等工具
   - 审核 Agent：判断是否需要人工确认
   - 总结 Agent：生成回复、处理记录和知识沉淀

3. 企业知识库 RAG
   支持上传 PDF、Markdown、Word、网页文档，做切分、向量化、召回、重排序、引用来源。

4. MCP 工具接入
   做几个 MCP Server，例如：
   
   - ticket-mcp：查工单、改状态
   - crm-mcp：查客户信息
   - db-mcp：安全查询业务数据
   - knowledge-mcp：检索知识库

5. 人工审批机制
   高风险动作不能直接执行，比如退款、删除数据、修改客户信息，必须进入人工确认。

6. Agent 可观测性
   前端展示每次 Agent 调用链路：
   
   - 调用了哪个 Agent
   - 检索了哪些文档
   - 调用了哪些工具
   - 花了多长时间
   - 是否命中人工审批
   - 最终是否解决

7. RAG/Agent 评估系统
   做一套测试集，评估：
   
   - 回答准确率
   - 引用是否真实
   - 工具调用是否正确
   - 是否误执行高风险动作
   - 工单解决率
   
   技术栈建议
   
   后端：
- FastAPI

- LangGraph

- LangChain

- PostgreSQL

- Redis

- Celery 或 RQ

- pgvector 或 Qdrant

- OpenAI / Qwen / DeepSeek 可配置
  
  前端：

- React + TypeScript

- 工单列表

- Agent 调用链路图

- 知识库管理

- 审批中心

- 评估看板



更具体一点：

- 基于 LangGraph 实现长流程、多分支 Agent 编排，支持工单分类、知识检索、工具调用、人工审批和结果总结。
- 构建企业知识库 RAG 管道，支持文档解析、切分、向量检索、重排序、引用溯源和回答 groundedness 评估。
- 设计 MCP 工具服务，将 CRM、工单系统、数据库查询封装为标准工具接口，降低 Agent 与业务系统耦合。
- 实现 AgentOps 观测面板，记录每次 Agent 的状态流转、工具调用、token 消耗、耗时、失败原因和人工介入点。
- 建立离线评估集，对检索准确率、回答正确性、工具调用准确性和高风险动作拦截率进行自动化评测。
