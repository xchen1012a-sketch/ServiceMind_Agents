# 项目命令规范

> AI 不猜命令。项目的真实命令来自构建清单、脚本和 `ai-spec.yaml`。

## 一、标准命令能力

项目按适用范围提供下列逻辑能力，具体命令由技术栈决定：

| 能力 | 用途 | 要求 |
|---|---|---|
| setup | 安装或恢复依赖 | 使用锁文件，可重复执行 |
| format | 格式化代码 | 范围可控，不顺带格式化全仓 |
| lint | 静态规则检查 | CI 与本地一致 |
| typecheck | 类型检查 | 不用跳过或宽松参数掩盖错误 |
| build | 生成可发布制品 | 非交互，失败码准确 |
| test | 默认测试入口 | 明确包含哪些测试层级 |
| test:unit | 单元测试 | 快速、隔离 |
| test:integration | 集成测试 | 声明依赖和数据隔离方式 |
| test:e2e | 端到端测试 | 声明环境、账号和清理策略 |
| run/dev | 本地运行 | 端口、依赖和停止方式明确 |
| migrate | 数据库迁移 | 环境显式、支持 dry-run/状态检查 |
| validate | 项目统一质量检查 | 组合适用的 lint/typecheck/test/build |

不适用的能力可以省略，但必须在项目画像中明确，而不是填入空命令冒充支持。

## 二、AI 执行规则

- 优先使用项目已有脚本，不绕过包管理器或构建工具直接调用内部实现。
- 执行前说明会产生的文件、网络、容器、数据库或外部服务副作用。
- 默认使用非交互和确定性参数；禁止依赖等待人工输入的隐藏步骤。
- 先运行受影响范围的快速检查，再根据风险运行更广回归。
- 不使用 `|| true`、吞错误或伪造退出码让流水线变绿。
- 不自动执行 deploy、publish、release、生产 migrate、批量通知和付费资源命令。
- 命令失败时保留关键错误、退出码和复现方式，不只报告“执行失败”。

## 三、多仓和 Monorepo

- 每条命令记录工作目录，禁止在协调父目录误执行仓库写操作。
- Monorepo 优先使用受影响模块能力，但发布候选按项目策略运行必要的全局门禁。
- 多仓任务分别记录各仓命令和结果，不把一仓通过当作全栈验证完成。

## 四、Git 与远端命令

- 本地 `git init` 只能在确认后的项目根执行；多子仓默认在各子项目执行，父目录只有 monorepo/聚合工程或用户显式指定时才执行。
- 无 Git 仓库接入前允许一次初始基线 commit：`git init`、`git add -A`、`git commit -m "chore: initial commit"`。该 commit 只能包含接入前已有文件，不得混入 `.ai-spec` 接入改动。
- 除初始基线 commit 外，AI 不自动 `git commit`。用户要求提交时，必须先给出 `git status --short`、提交范围、提交信息、验证结果和未验证项。
- 远端操作默认只给命令建议，不自动执行。执行 `git remote add`、创建远端仓库、`git push`、`git push -u`、分支保护或协作者设置前，必须确认 remote 名称、URL/仓库名、仓库可见性、目标分支、凭证来源和是否推送当前分支。
- 远端提交默认只提交**有代码变更的项目仓库**。多子仓场景下，父目录 `.specforge.json` 或父级 Bootstrap 规则不默认提交/推送；AI 必须告知用户“默认建议提交子仓，父目录如需提交请单独确认”。用户明确要求父目录也提交时，才按父目录 Git 归属单独处理。
- **提交前扫描门禁**：任何 `git commit` 或远端 `git push` 前，必须先在目标仓库运行 `scripts/git-preflight.ps1` 或等价扫描；多子仓逐仓扫描。扫描报告至少包含 `git status --short`、本次待提交文件、被过滤文件、疑似密钥/本地配置/构建产物检查结果。扫描未通过时禁止提交和推送。
- 提交前必须过滤非项目代码和本地配置。默认不得提交：`.env*`、真实密钥/Token、IDE/编辑器本地配置（`.idea/`、`.vscode/` 等，除项目明确共享的推荐配置）、本机路径配置、日志、缓存、构建产物（`dist/`、`build/`、`target/`、`.next/`、`coverage/`）、依赖目录（`node_modules/`、`vendor/`）和临时文件。
- 可提交范围默认限于：项目源码、测试、必要项目文档、锁文件、构建/脚本配置、CI 配置，以及该子仓内已确认需要版本化的 `.ai-spec` 规则。父目录索引和 Bootstrap 规则除非用户确认，否则只报告不纳入默认提交。
- 禁止把凭证、Token、个人账号信息写入文档、计划、命令记录或 Git remote URL。需要认证时使用本机凭证管理器、环境变量或平台 CLI 已登录状态。
- 禁止 `git push --force`。唯一例外：用户明确指定自己的临时分支、确认无他人协作，并说明回滚方式；优先使用 `--force-with-lease`。
- 多仓推送必须逐仓确认和报告，不把一个子仓的 remote/upstream 复用于其它子仓。

## 五、命令变更

新增或修改标准命令时同步更新：

1. 构建清单或任务脚本；
2. `ai-spec.yaml`；
3. CI/CD；
4. 开发文档；
5. 对应验证测试。
