# 轻量交付规则

用于小改动、局部 Bug 修复、视觉/文案/配置调整。只有本次差异实际改变业务不变量、公开契约、权限边界、数据迁移或安全边界时，才升级读取完整规范。

## 最低要求

- 说明改了什么，范围控制在哪些文件。
- 给出实际验证命令或可复查的检查方式。
- 明确未验证项，不用“应该可以”代替结果。
- 给出回滚方式：还原文件、撤销配置、或回退分支。

## 短版输出

L0/L1/L2 默认短版输出，最多三段：

```text
完成情况：
- ...

验证：
- ...

注意：
- ...
```

禁止流水账：不要复述每个工具调用、不要罗列所有读过的规范文件、不要在无风险小改动里写长篇背景。

表格最多 3 列；风险、依据、建议这类长内容优先用编号列表，不塞进宽表格。

## 不需要升级的例子

- typo、注释、格式化。
- 单个按钮文案、颜色、间距、图标替换。
- 不改变行为的常量命名或小范围重命名。

## 必须升级的例子

- 业务逻辑、状态机、金额、库存、KPI。
- API schema、事件、权限、认证、租户、数据范围。
- 数据库 schema、迁移、批量写入。
